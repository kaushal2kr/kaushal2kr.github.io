CCNA + NOC Interview Notes

Networking Fundamentals for NOC, IT Support & Cybersecurity Interviews

These notes explain the core networking concepts needed for CCNA basics, NOC roles, IT support, SOC analysis, cybersecurity fundamentals, and packet analysis.

The goal is not only to memorize definitions. The goal is to understand how devices communicate, how packets move, how headers are added, and how to troubleshoot issues step by step.

Best Study Order

Start from the basic building blocks. Do not jump directly into routing or cybersecurity without understanding packet, frame, IP, MAC, ARP, DNS, DHCP, and gateway.

Recommended Learning Flow

  1. Understand what data, packet, frame, header, encapsulation, and decapsulation mean.
  2. Understand OSI model and TCP/IP model from practical troubleshooting point of view.
  3. Understand IP address, MAC address, subnet mask, default gateway, and DNS server.
  4. Understand DHCP: how a device gets IP automatically.
  5. Understand DNS: how website names become IP addresses.
  6. Understand ARP: how IP address is mapped to MAC address inside LAN.
  7. Understand what happens when you open a website.
  8. Understand switch, router, NAT, TCP handshake, TLS, and HTTP/HTTPS.
  9. Learn basic troubleshooting commands used by NOC and IT support engineers.

Core Building Blocks

These terms are the base of networking. If these are clear, OSI, TCP/IP, Wireshark, routing, and SOC investigation become easier.

📦

Data

Data is the original information created by an application. Example: a browser request, chat message, file, image, or video stream.

📨

Packet

A packet is data with Layer 3 information, mainly source IP and destination IP. Routers use packets to forward traffic between networks.

🧱

Frame

A frame is data with Layer 2 information, mainly source MAC and destination MAC. Switches use frames inside a LAN.

🏷️

Header

A header is extra control information added before data. Each layer adds its own header so devices can understand how to handle the data.

Encapsulation and Decapsulation

Encapsulation means headers are added while data goes down the network stack. Decapsulation means headers are removed when data reaches the receiver.

Encapsulation Flow

When your laptop sends data, the application creates the data. Then every layer adds its own information.

Application Data
↓ Transport Layer adds TCP/UDP header = Segment
↓ Network Layer adds IP header = Packet
↓ Data Link Layer adds MAC header/trailer = Frame
↓ Physical Layer sends bits/signals through cable or Wi-Fi

Decapsulation Flow

On the receiving side, the reverse happens. The receiver removes each header and gives the original data to the application.

Bits/signals received
↓ Frame is checked at Data Link Layer
↓ Packet is processed at Network Layer
↓ Segment is processed at Transport Layer
↓ Original data reaches the application

OSI Model vs TCP/IP Model

OSI is useful for troubleshooting and understanding concepts. TCP/IP is the practical model used on the internet.

OSI Layer What It Does Examples Troubleshooting View
Layer 7 - Application User-facing network services HTTP, HTTPS, DNS, SMTP Website, app, DNS, login, service issue
Layer 6 - Presentation Format, encryption, encoding TLS/SSL, compression, encoding Certificate, encryption, format issue
Layer 5 - Session Session creation and management Login session, connection session Session timeout or broken session
Layer 4 - Transport End-to-end communication TCP, UDP, port numbers Port blocked, TCP handshake fail, retransmission
Layer 3 - Network Logical addressing and routing IP address, router, ICMP Wrong IP, gateway issue, routing issue
Layer 2 - Data Link Local network delivery MAC, switch, VLAN, ARP VLAN issue, MAC learning issue, STP issue
Layer 1 - Physical Physical signal transmission Cable, port, Wi-Fi signal, LED Cable unplugged, bad port, no link light

IP Address, MAC Address, Subnet Mask and Gateway

These four things are very important for real troubleshooting.

IP Address

IP address is a logical address used to identify a device on a network. Routers use IP addresses to move packets between different networks.

MAC Address

MAC address is a physical Layer 2 address of a network interface. Switches use MAC addresses to forward frames inside the same LAN.

Subnet Mask

Subnet mask tells which part of the IP address is network and which part is host. It helps the device decide whether destination is local or remote.

Default Gateway

Default gateway is the router IP used when the destination is outside the local network. Without gateway, internet or remote network access will fail.

Local vs Remote Communication

If source and destination are in the same network, the device uses ARP to find the destination MAC address and sends traffic directly inside the LAN.

If destination is in another network, the device sends the frame to the default gateway MAC address. Then the router forwards the packet toward the remote network.

DHCP, DNS and ARP

These three protocols are used almost every time a device joins a network and opens a website.

DHCP - Dynamic Host Configuration Protocol

DHCP automatically gives IP address, subnet mask, default gateway, DNS server, and lease time to a device. Without DHCP, users may need to configure IP settings manually.

DNS - Domain Name System

DNS converts human-readable domain names like google.com into IP addresses. Without DNS, users would need to remember IP addresses instead of names.

ARP - Address Resolution Protocol

ARP finds the MAC address for a known IP address inside the same local network. It is required because switches forward frames using MAC addresses.

DHCP DORA Process

DHCP works through four main steps called DORA.

Discover

Client broadcasts a message: “Is there any DHCP server available?” This is broadcast because the client does not yet have an IP address.

Offer

DHCP server replies with an available IP address and network settings.

Request

Client requests to use the offered IP address.

Acknowledgement

DHCP server confirms the lease. Now the client can use that IP address.

DNS Records

DNS has different record types for different purposes.

Record Meaning Example Use
A Maps domain to IPv4 address example.com → 93.184.216.34
AAAA Maps domain to IPv6 address example.com → IPv6 address
CNAME Alias for another domain name www.example.com → example.com
MX Mail exchange server Used for email delivery
TXT Text-based verification/security info SPF, DKIM, domain verification
NS Name server record Tells which DNS servers are authoritative
PTR Reverse DNS lookup IP address → domain name

What Happens When We Open a Website?

This is one of the most important interview questions for networking, IT support, NOC, SOC, and cybersecurity roles.

Step 1: User enters website name

User types a domain name like example.com in the browser.

Step 2: DNS resolution

Browser/system checks cache first. If IP is not found, it queries DNS server to get the IP address of the website.

Step 3: ARP for gateway MAC

If the website IP is outside the local network, the device needs the MAC address of the default gateway. It uses ARP to find the gateway MAC.

Step 4: Frame goes to switch

The device sends an Ethernet frame to the switch. The destination MAC is the gateway MAC.

Step 5: Router forwards packet

The router removes the Layer 2 frame, reads the destination IP, and forwards the packet toward the internet.

Step 6: NAT may happen

In home/office networks, private IP is translated into public IP using NAT.

Step 7: TCP handshake

For HTTPS websites, TCP connection is created using SYN, SYN-ACK, and ACK.

Step 8: TLS handshake

Browser and server establish encryption so HTTPS traffic stays secure.

Step 9: HTTP request and response

Browser sends HTTP/HTTPS request. Server sends back HTML, CSS, JavaScript, images, and other resources.

Step 10: Browser renders webpage

Browser processes the received content and displays the webpage to the user.

Switch, Router, NAT, TCP and TLS

These are practical terms you should explain clearly in interviews.

Term Simple Meaning Practical Role
Switch Layer 2 device Forwards frames inside LAN using MAC address table
Router Layer 3 device Forwards packets between different networks using IP routing
NAT Network Address Translation Translates private IP to public IP for internet access
TCP Reliable transport protocol Uses handshake, sequencing, acknowledgement, retransmission
UDP Fast connectionless protocol Used in DNS, video call, gaming, streaming where speed matters
TLS Encryption layer for secure communication Used in HTTPS to protect data between browser and server

NOC Troubleshooting Mindset

NOC engineers troubleshoot logically. They do not guess randomly.

Start from Layer 1

Check power, cable, port LED, Wi-Fi signal, link status, and physical connection.

Then check Layer 2

Check MAC address, switch port, VLAN, STP, and local connectivity.

Then check Layer 3

Check IP address, subnet mask, gateway, routing, and ping.

Then check Layer 4 and Application

Check TCP/UDP ports, firewall, DNS, application service, and server response.

Interview Answer: What Happens When We Open a Website?

When we type a website name in the browser, the system first checks whether the IP address is already available in cache. If not, it sends a DNS query to resolve the domain name into an IP address.

After getting the destination IP, the device checks whether the destination is in the same local network or a remote network. If it is remote, the packet must go to the default gateway. To send the frame to the gateway, the device uses ARP to find the gateway MAC address.

Then the Ethernet frame goes to the switch. The switch forwards it using the MAC address table. The router receives it, removes the Layer 2 frame, checks the destination IP, and forwards the packet toward the internet. NAT may translate the private IP into a public IP.

For an HTTPS website, TCP three-way handshake happens first. After that TLS handshake happens to create a secure encrypted session. Then the browser sends the HTTP request and the server replies with webpage content like HTML, CSS, JavaScript, images, and other resources.

Finally, the browser renders the webpage. From troubleshooting point of view, failure can happen at many points such as DNS failure, ARP/gateway issue, IP issue, routing issue, firewall issue, TCP port issue, TLS certificate issue, or application server issue.

Quick Revision Summary

Use this section for last-minute interview revision.

  • Switch: forwards frames using MAC address.
  • Router: forwards packets using IP address.
  • ARP: finds MAC address for an IP inside LAN.
  • DNS: converts domain name into IP address.
  • DHCP: automatically assigns IP settings to clients.
  • Packet: Layer 3 unit with IP header.
  • Frame: Layer 2 unit with MAC header and trailer.
  • TCP: reliable, connection-oriented protocol.
  • UDP: faster, connectionless protocol.
  • NAT: translates private IP into public IP.
  • Gateway: router used to reach other networks.
  • Subnet mask: identifies network and host portion.
